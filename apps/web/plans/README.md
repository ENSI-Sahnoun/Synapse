## Recommended execution order

1. **001** first — purely mechanical, zero behavior risk, clears the noisiest finding class.
2. **003** next — single global CSS block, independent of everything else, high blast radius (affects every user with reduced-motion enabled) so worth landing early.
3. **002** and **004** — independent of each other and of 001/003; both touch motion/react components only. Can run in either order or in parallel worktrees.
4. **005** last — touches `motion/react` `AnimatePresence`/`layout` patterns; no hard dependency on the others, but doing it last means the executor can copy the by-then-familiar `motion/react` import conventions from 002/004 if useful (not required — plan 005 is self-contained either way).

No plan depends on another completing first; the ordering above is a suggestion for review clarity, not a hard sequencing requirement.
